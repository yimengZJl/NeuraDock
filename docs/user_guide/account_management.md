# 账号管理

## 添加账号

1. 从侧边栏导航到**账号**页面
2. 点击**添加账号**按钮
3. 填写账号表单：

| 字段 | 必填 | 描述 |
|-----|------|------|
| 名称 | 是 | 显示名称（通常是邮箱地址） |
| 服务商 | 是 | 服务提供商（AnyRouter、AgentRouter） |
| Cookies | 是 | JSON 格式的认证 cookies |
| API 用户 | 是 | API 用户标识符（服务商特定） |
| 自动签到 | 否 | 启用每日自动签到 |
| 签到时间 | 否 | 自动签到时间（24小时制） |

4. 点击**保存**创建账号

## 获取 Cookies

1. 在浏览器中登录到服务商网站
2. 打开开发者工具（F12 或 Cmd+Option+I）
3. 转到 **Application** 标签 → **Cookies**
4. 复制所需的 cookie 值
5. 格式化为 JSON：

```json
{
  "session": "your_session_token_here"
}
```

> **注意**：只需提供 `session` cookie，WAF 绕过所需的其他 cookies（如 `cf_clearance`）会在运行时自动获取。

## 编辑账号

1. 在账号页面找到账号卡片
2. 点击卡片上的 **⋮**（更多）菜单
3. 选择**编辑**
4. 修改所需字段
5. 点击**保存**

## 删除账号

1. 找到账号卡片
2. 点击 **⋮** 菜单
3. 选择**删除**
4. 确认删除

**警告**：删除账号也会删除其签到历史和余额记录。

## 启用/禁用账号

- 切换账号卡片上的开关以启用/禁用
- 禁用的账号不会出现在批量操作中
- 禁用账号的自动签到会暂停

## 批量导入

使用 JSON 一次导入多个账号：

1. 在账号页面点击**导入**按钮
2. 粘贴 JSON 数组：

```json
[
  {
    "name": "account1@example.com",
    "provider": "anyrouter",
    "cookies": {
      "session": "token1"
    },
    "api_user": "user1"
  },
  {
    "name": "account2@example.com",
    "provider": "agentrouter",
    "cookies": {
      "session": "token2"
    },
    "api_user": "user2"
  }
]
```

3. 点击**导入**
4. 查看结果

## 批量更新

更新多个现有账号的凭证：

1. 在账号页面点击**批量更新**
2. 粘贴包含更新凭证的 JSON
3. 可选择启用"不存在则创建"
4. 点击**更新**

## 导出账号

1. 选择要导出的账号（或导出全部）
2. 点击**导出**按钮
3. 选择是否包含凭证
4. 保存 JSON 文件

**安全提示**：包含凭证的导出文件含有敏感数据，请安全存储。
