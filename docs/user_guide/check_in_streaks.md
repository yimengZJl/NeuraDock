# 签到记录与统计

签到记录功能提供详细的签到历史、连续签到统计和可视化分析，帮助你跟踪签到习惯和收益情况。

![签到记录界面](../assets/app-streak-zh.png)

## 功能概览

- 📊 **连续签到统计** - 当前连续天数、最长连续记录
- 📅 **日历视图** - 直观显示每月签到情况
- 📈 **趋势分析** - 30天签到趋势图表
- 💰 **收益追踪** - 每次签到的余额增量
- 🎯 **按账号查看** - 支持查看单个账号或所有账号

## 页面导航

1. 从侧边栏点击 **签到记录** 进入页面
2. 使用右上角的账号选择器切换视图：
   - **所有账号** - 查看所有账号的签到概览
   - **单个账号** - 查看特定账号的详细记录

选择特定账号后，可以查看详细的签到记录和统计。

### 账号概览面板

左侧面板显示账号的关键统计：

#### 基本信息
- 账号名称
- 服务提供商
- 最后签到日期

#### 连续签到统计
- **当前连续** 🟠 - 当前连续签到天数

- **总签到天数** 🔵 - 累计签到总天数

## 连续签到规则

### 连续计算规则

1. **开始连续** - 成功签到后，连续计数从 1 开始
2. **保持连续** - 每天都签到，连续天数 +1
3. **中断连续** - 某天未签到，连续天数归零
4. **最长记录** - 自动追踪历史最长连续天数

### 连续签到示例

```
日期:    1日  2日  3日  4日  5日  6日  7日
签到:    ✓    ✓    ✓    ✗    ✓    ✓    ✓
连续:    1    2    3    0    1    2    3
最长:    1    2    3    3    3    3    3
```

## 签到率计算

签到率 = (本月已签到天数 / 本月总天数) × 100%

**示例**：
- 本月 30 天
- 已签到 25 天
- 签到率 = (25 / 30) × 100% = 83.3%

### 跨月连续

连续签到跨月计算规则：
- 12月31日签到 + 1月1日签到 = 连续保持
- 任何一天断签 = 连续中断

## 常见问题

### Q: 为什么连续天数归零了？

**A**: 可能的原因：
- 某天忘记签到（手动或自动）
- 签到失败（查看失败日期的详情）
- 应用未运行导致自动签到未执行

### Q: 签到记录多久会被清除？

**A**: 签到记录永久保存在本地数据库中，不会自动清除。

### Q: 可以修改历史签到记录吗？

**A**: 目前不支持手动修改历史记录。所有记录由系统自动生成。

### Q: 为什么有些日期没有额度增量？

**A**: 可能的原因：
- 签到失败
- 服务商当天没有提供收益
- 获取余额信息失败

### Q: 趋势图为什么只显示 30 天？

**A**: 为了保持图表清晰易读，默认显示最近 30 天。未来可能支持自定义时间范围。

### Q: 可以导出签到记录吗？

**A**: 目前不支持导出功能，计划在未来版本中添加。
